総評

  • 基本検索(キーワード / 密ベクトル / 地理空間 / ファセット / マルチモーダル / ソート等)は4製品すべて対応しており横並び。
  • Elastic と OpenSearch は機能網羅性が高く、学習済み疎ベクトル(SPLADE系)、重み付き線形結合ハイブリッド、SQL+PPL、Explain API までフルサポート。
  • Azure AI Search は学習済み疎ベクトル、重み付き線形結合、グラフトラバーサル、Explain API すべて非対応または限定対応で、検索方式の柔軟性が4製品中最も低い。
  • Vertex AI Search は重み付き線形結合とグラフトラバーサルが非対応、追加クエリ言語も独自DSLのみ。
  • グラフトラバーサル / 関係性検索は Elastic のみ対応で、ナレッジグラフ的な探索を必要とする要件で唯一の選択肢になる。

総評

  • Elastic は HNSW に加え独自の DiskBBQ を持ち、ストレージ最適化と精度のバランスを取りやすい。

  • バイナリ量子化・スカラー量子化は Elastic / Azure / OpenSearch が対応、Vertex のみ非対応。コスト最適化の選択肢で Vertex が劣後する。
  • フィルタ併用 ANN 高速化(ACORN 等)は Azure のみ非対応で、メタデータフィルタ多用シナリオでは Azure が遅延・精度面で不利。
  • 最大次元数は OpenSearch 16,000 が最大、他3製品は 4,096(Vertex は明示上限非公開)。
  • インデックスアルゴリズム選択肢は OpenSearch(HNSW / IVF / IVF-PQ / Brute Force)が最も豊富。
