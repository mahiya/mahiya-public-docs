# ELSER

## 概要

ELSER（Elastic Learned Sparse EncodeR）は、Elastic社が独自に開発・学習したスパースベクトルベースのセマンティック検索モデルである。2023年6月にElasticsearch 8.8のテクニカルプレビューとして初公開され、2023年11月のElasticsearch 8.11でGA（一般提供）となった。従来のキーワードマッチ（BM25）では捉えきれない文脈や意図を理解し、ユーザーのクエリを意味的に拡張（text expansion）することで検索精度を向上させる。アーキテクチャにはSPLADE（Sparse Lexical and Expansion Model）を採用し、BERTのマスク言語モデルヘッドを活用してトークンごとの活性化分布を生成する。パラメータ数は約110Mとコンパクトながら、BEIRベンチマークではBM25に対してNDCG@10で平均17〜18%の改善を達成。ファインチューニング不要のゼロショット検索を実現し、Elasticsearch上でワンクリックで導入できる手軽さが特徴である。Consensus、Georgia State Universityなどの本番環境で採用実績があり、Hugging Face MTEBリーダーボードの検索タスク（250Mパラメータ以下クラス）でもトップ10にランクインしている。現在ELSER v3が開発中であり、LLM生成データの活用やGPU推論への対応が進められている。

## 技術者向けポイント

### 1. SPLADEアーキテクチャとスパースベクトル表現

- BERTのマスク言語モデル（MLM）ヘッドを拡張し、入力テキストの各トークンに対する活性化分布を集約してスパースベクトルを生成する仕組みである
- ボキャブラリサイズ（30,522次元）のうち、実際に活性化されるのはごく一部であり、Elasticsearchの転置インデックス（Lucene）で効率的に格納・検索できる
- 密ベクトル（Dense Vector）と比較して、インデックスサイズがテキストインデックスとほぼ同等であり、近似最近傍探索（ANN）より省メモリである
- トークンは単なる同義語ではなく、学習された関連性の高い共起語であり、低重みトークンを除去するとPrecisionも含む全指標が低下することから、分散表現的に機能している

### 2. クエリ最適化：トークンプルーニングとリスコアリング

- ELSER v2では、コーパス内で想定頻度の5倍以上出現する高頻度トークンを検索フェーズから除外し、上位50件をフルトークンでリスコアリングする戦略を採用
- この最適化により、NDCG@10を平均+0.07%とほぼ劣化なしで、検索速度を平均3.4倍高速化した（block-max WAND有効時は最大8.4倍）
- Elasticsearch 8.13.0以降では`text_expansion`クエリにトークンプルーニングが自動統合されており、追加設定なしで適用される
- BM25のIDF重みと異なり、ELSERのトークン重みは頻度と独立しているため、コーパス固有の頻度情報を活用することがブレークスルーとなった

### 3. 学習設計：蒸留とFLOPS正則化

- クロスエンコーダ蒸留により、教師モデル（monot5 3bとMiniLM L-6のアンサンブル）のスコアマージンをMSE損失で学習する手法を採用
- FLOPS正則化は推論コストをペナルティとして課すことで、トークンの99%が最初の50,000バッチ（ウォームアップ期間）で刈り込まれ、汎化性能も同時に向上する
- ELSER v2ではモデル選択にパレート最適化を導入し、検索精度を約1%犠牲にして検索コストを25%以上削減するトレードオフを実現
- Intel x86-64向け量子化モデル（ハイブリッド動的量子化）により推論速度を1.7〜2.2倍改善しつつ、精度劣化を最小限に抑えている

### 4. デプロイメントとインフラストラクチャ

- 推奨デプロイ方法はInference APIを通じたサービス利用であり、`semantic_text`フィールドタイプと組み合わせることでインデックス作成からクエリまでシームレスに動作する
- Elastic Inference Service（EIS）ではRay/Ray Serveフレームワーク上でKubernetes＋NVIDIA GPUプールを活用し、GPU推論による高速な埋め込み生成を提供
- MLノードの最小サイズは4GB（Elastic Cloud Hosted）であり、Adaptive Allocationsによるオートスケーリングで負荷に応じた動的リソース調整が可能
- エアギャップ環境でもHTTPサーバーまたはファイルベースでモデルアーティファクトを配布でき、閉域ネットワークへのデプロイにも対応している

## ビジネス側向けポイント

### 1. ゼロショット導入による即時価値提供

- ファインチューニングやドメイン固有の学習データ準備が不要で、Kibana上のボタンクリックだけでセマンティック検索を有効化できる
- 従来のBM25キーワード検索と比較して、12種類のベンチマークのうち10勝1分1敗（平均17〜18%のNDCG@10改善）を達成しており、検索精度の向上が実証済みである
- RAG（Retrieval-Augmented Generation）アプリケーションの基盤として、LLMに高品質なコンテキストを提供し、ハルシネーションの低減に寄与する
- ConsensusやGeorgia State Universityなど実運用での採用事例があり、エンタープライズ品質が検証されている

### 2. コスト効率とリソース最適化

- 110Mパラメータのコンパクトなモデルであり、数十億パラメータの大規模モデルと比較してインフラコストが大幅に低い
- スパースベクトル方式のため密ベクトルインデックスよりもストレージ使用量が少なく、既存のElasticsearchインフラ上で稼働する
- ELSER v2の最適化版はインジェスト速度が最大26 docs/sとv1比で90%向上しており、大量データの処理効率が改善されている
- EIS（Elastic Inference Service）利用時はトークンベースの課金であり、自前でMLノードを管理する必要がないためTCOを抑制できる

### 3. プラットフォームの柔軟性と拡張性

- ハイブリッド検索（ELSER＋BM25のスコア統合やReciprocal Rank Fusion）に対応しており、検索精度をさらに向上させる組み合わせ運用が可能
- Inference APIを通じてOpenAI等のサードパーティモデルと共存でき、ベンダーロックインを避けつつ最適なモデルを選択できる
- LangChain・LlamaIndexとの統合がElasticsearch 8.11以降で公式サポートされ、AIアプリケーション開発エコシステムとシームレスに連携する
- Elasticsearch Serverlessでの動作にも対応しており、クラウドネイティブな運用からオンプレミスのエアギャップ環境まで幅広い構成に適用可能

### 4. 将来のロードマップと継続的投資

- ELSER v3が現在開発中であり、LLM生成データを活用した高度な学習レシピや新しいトレーニング戦略が導入される予定である
- GPU推論への本格対応により、大規模インジェストワークロードの処理性能がさらに向上する見込みである
- 将来的にはELSER via EISが`semantic_text`フィールドのデフォルト設定となり、明示的な推論ID指定なしでセマンティック検索を利用できるようになる
- MTEBリーダーボードでの継続的な評価公開により、競合モデルとの透明な比較が可能であり、モデル選定の判断材料として活用できる
