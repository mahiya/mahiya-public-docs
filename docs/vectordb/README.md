## エグゼクティブサマリー

ベクトルデータベース市場は2025年時点で約25億ドル規模に達し、LLMとRAGパイプラインの急速な普及を背景に年率27%超で成長中である。本レポートでは市場での採用実績・認知度・GitHubスター数・資金調達規模を根拠として選定したトップ5プロダクトを、27の比較項目にわたって徹底比較した。

## 1. プロダクト概要

### Pinecone
- **開発元**: Pinecone Inc.（米国）、評価額 20億ドル超（2025年）
- **形態**: クローズドSaaS専用、セルフホスト不可
- **ポジション**: マネージドベクトルDB市場のパイオニア。RAGワークロードのデファクトスタンダードとして最多言及。
- **公式URL**: https://www.pinecone.io/

### Milvus / Zilliz Cloud
- **開発元**: Zilliz（米国/中国）、Apache Software Foundation
- **形態**: Apache 2.0 OSSエンジン + Zilliz Cloudマネージドサービス
- **ポジション**: GitHubスター40,000+、10,000社以上の本番採用。NVIDIA・Salesforce・eBayなど超大規模エンタープライズが採用。
- **公式URL**: https://milvus.io/ / https://zilliz.com/

### Qdrant
- **開発元**: Qdrant Solutions GmbH（ドイツ）、Series B $50M調達（2026年3月）
- **形態**: Apache 2.0 OSS + Qdrant Cloudマネージドサービス
- **ポジション**: Rust実装による最高クラスの性能・コスト効率。Tripadvisor・HubSpot・Canvaが採用。GigaOm Radar 2025「Leader」。
- **公式URL**: https://qdrant.tech/

### Weaviate
- **開発元**: Weaviate B.V.（オランダ）、Series C $200M評価額
- **形態**: Apache 2.0（実際はBSD 3-Clause）OSS + Weaviate Cloud（旧WCS）
- **ポジション**: ハイブリッド検索・マルチテナント・埋め込みモジュール統合が特徴。エンタープライズAIプラットフォームへ進化中。
- **公式URL**: https://weaviate.io/

### pgvector
- **開発元**: OSS Community（PostgreSQL拡張）
- **形態**: PostgreSQL License（MITと同等の寛容ライセンス）のOSS拡張、v0.8.2（2026年2月）
- **ポジション**: PostgreSQLの広大なエコシステムを活用した「追加コストゼロ」のベクトル検索。PostgreSQL採用率55.6%（2025年）の追い風。
- **公式URL**: https://github.com/pgvector/pgvector

## 4. ユースケース別推奨プロダクト

### ユースケース 1: スタートアップ・MVP開発
**推奨: Qdrant または Pinecone**

- Pinecone: インフラ管理ゼロで最速立ち上げ。フリーティアで検証後、企業の成長とともにプランをスケール。
- Qdrant: Docker 1コマンドでローカル起動。フリーティア（1GBメモリ制限）で開発・検証し、Qdrant Cloudへシームレス移行可能。

### ユースケース 2: 大規模エンタープライズRAG（数十億ベクトル超）
**推奨: Milvus / Zilliz Cloud**

- 数十億〜兆規模のベクトルに対応する唯一の成熟OSS。
- NVIDIA GPU加速でインデックス構築50倍高速化。
- DiskANNによりSSD活用でメモリコスト削減。
- Zilliz Cloudで運用コストを低減しつつエンタープライズSLAを確保。

### ユースケース 3: 高性能・低コストのセルフホスト運用
**推奨: Qdrant**

- Rust実装でp99 < 2msのレイテンシ・10,000 QPS超のスループットを実現。
- セルフホスト時のコストが最安値クラス（他社比で月額1/10以下の事例あり）。
- Apache 2.0でベンダーロックインなし。シングルバイナリでDocker Compose起動可能。

### ユースケース 4: マルチモーダル・AIネイティブアプリケーション
**推奨: Weaviate**

- text2vecモジュールでOpenAI/Cohere/HuggingFaceの埋め込みを自動生成・統合。
- ColBERT Late Interactionで高精度検索を実現。
- ネイティブマルチテナントで数百万テナントに対応（SaaS型AI製品向け）。
- SOC2 + HIPAA + ISO27001 + GDPRのフルコンプライアンス認証。

### ユースケース 5: PostgreSQLスタック上でベクトル検索を追加
**推奨: pgvector**

- PostgreSQL環境にEXTENSION 1行で追加。追加費用ゼロ。
- ACID保証・JOIN・トランザクション等のRDB機能をそのまま活用。
- Instacartは Elasticsearch からの移行でコスト80%削減。
- 規模の目安: 1,000万〜3,000万ベクトルまでの中規模が実用的（それ以上は専用DBを検討）。

## 6. 主要リスク・注意点

### Pinecone
- **ベンダーロックイン**: クローズドSaaSでデータ移行のコストが高い
- **コスト**: 大規模利用ではセルフホスト比2倍以上になることが多い
- **インデックス非公開**: HNSW/IVF等のアルゴリズム選択ができず、量子化も非対応
- **最新動向**: 2025年にCEO交代・大口顧客Notionの離脱が報告されており、企業継続性に一部懸念

### Milvus
- **運用複雑性**: etcd + MinIO + Kafka/Pulsar + 多数マイクロサービスでK8s専任エンジニアが必要
- **学習曲線**: 設定パラメータ500以上、コンポーネント障害診断が難しい
- **レイテンシ**: 単一クエリのp99がQdrantの約3倍（5ms vs 2ms）

### Qdrant
- **マルチテナント上限**: 推奨上限1,000テナント未満（大規模SaaSでは制約になる場合がある）
- **ISO27001未取得**: Pinecone・Milvus・Weaviateより認証が少ない
- **インデックス多様性**: HNSWのみ（DiskANN・IVF等は非対応）

### Weaviate
- **スパースベクトル未対応**: SPLADEベースのハイブリッド検索には対応しておらず（GitHub Issue #4019がオープン）
- **フリーティアなし**: マネージドSaaSの試用が14日間限定
- **レイテンシ**: Qdrantに比べてp99が約5〜10倍高い報告あり

### pgvector
- **スケール限界**: 実用的な上限は数千万ベクトル（それ以上はパフォーマンス劣化）
- **機能不足**: 量子化（SQ/PQ）・高度なフィルタリング・ColBERT等の先進機能は非対応
- **スループット**: 大規模同時クエリでは専用DBの1/10以下のQPSになることがある
- **依存性**: ハイブリッド検索にParadeDB等の外部拡張が必要

## 7. 市場動向まとめ

- **価格競争の激化**: Pineconeが2025年に料金30%値下げ、Turbopufferが2026年2月にクエリ料金94%値下げを実施。コモディティ化が加速。
- **ハイブリッド検索の標準化**: BM25+ベクトルのハイブリッド検索はすでに事実上の標準機能となり、Late Interaction（ColBERT）の普及も進んでいる。
- **M&A予測**: 2026年末までに主要クラウドプロバイダによるベクトルDB企業の2〜3件の買収が予測されている。
- **エージェントAI対応**: Weaviate・Milvus・Qdrantが2026年にAIエージェント向け機能（GraphRAG・マルチホップ検索）を強化中。
- **新興勢力**: Turbopufferが Cursor・Notion・Linear・Anthropic に採用されており、次の比較対象候補として急浮上。
